# 🚀 Ver2.0 更新日志

终于把 **Ver2.0** 整理出来了。  
这一版不只是修修补补，而是把整个项目往更完整的研究框架方向推进了一步。

相比 Ver1.0，Ver2.0 主要补上了几个我一直觉得很重要的部分：**执行层更真实、实验更容易复现、分析能力更完整、工程结构也更稳了。**

---

## ✨ 新增内容

### 📈 更真实的回测执行
这一版补强了执行层，不再只是比较理想化的回测。

新增/增强了：

- 多种执行价格模式  
  - `same_close_idealized`
  - `next_open`
  - `next_close`
- 可配置滑点 `slippage_bps`
- 更清晰的交易成本处理
- 执行假设单独输出，方便复盘

---

### 🧩 更稳健的数据缺失处理
针对 `volume`、`amount`、`turnover_rate` 这些字段，增加了更灵活的缺失值处理方式：

- `zero`
- `ffill`
- `drop`
- `keep_nan`

同时也支持自动生成缺失标记列，比如：

- `volume_was_missing`
- `amount_was_missing`
- `turnover_rate_was_missing`

这样后面做分析时会更清楚数据到底是“真没有”还是“被补过”。

---

### 🔁 更完整的实验复现能力
这一版把实验记录这块补强了不少。

现在会记录更多运行信息，比如：

- git commit
- Python / 依赖版本
- 数据文件哈希
- 随机种子
- 时间戳等 metadata

也更方便后续复现实验、回头查历史结果。

---

### 🧠 新增一些研究分析模块
Ver2.0 新加了几块我后面会继续用来做研究的模块：

- `calibration.py`
- `explainer.py`
- `regime.py`
- `rolling_trainer.py`
- `html_reporter.py`
- `run_manifest.py`
- `repro.py`
- `compare_runs.py`

主要对应这些方向：

- 概率校准
- 模型解释
- 市场状态分析
- 滚动训练
- HTML 报告
- 实验记录 / 对比 / 复现

---

## 🛠 工程化改进

这一版也顺手把工程结构补了一轮：

- 加了 `config_schema.py` 做配置校验
- 加了结构化日志
- 加了启动检查
- 补了 `tests/`
- 加了 GitHub Actions CI

现在整个项目至少不像之前那么“纯脚本感”了，后面继续改也更安心一些。

---

## 🐞 修复 & 优化

这次也顺手修了一些坑，包括：

- 执行可行性检查逻辑
- 上市天数 / 成交校验相关问题
- 缺少 `amount` / `volume` 字段时的兼容性
- CLI 导入阶段的一些环境处理问题
- 一些依赖和 CI 配置问题

---

## ✅ 这一版想解决什么

简单说，Ver2.0 主要想解决几件事：

- 让回测别太理想化
- 让实验结果更容易复现
- 让研究分析不只停留在“跑出收益曲线”
- 让项目结构更适合继续往下迭代
